Sobre esta Aplicação

1. Arquitetura MVT do Django

O Django utiliza o padrão MVT (Model-View-Template):

  • Model (M): Gere a base de dados. Define as tabelas e as relações (ex: models.py).
  • View (V): É o "cérebro". Recebe o pedido do utilizador, vai buscar os dados ao Model e envia-os para o Template (ex: views.py).
  • Template (T): É a interface do utilizador. O ficheiro HTML que junta o design com os dados enviados pela View (ex: sobre.html).

Fluxo: Utilizador faz pedido -> Django URL encaminha -> View processa -> View pede dados ao Model -> View envia dados para o Template -> Utilizador vê a página final.
